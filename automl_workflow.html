User drops CSV path + natural language dataset description into Textual Input widgets. Pandas loads the DataFrame. LangGraph initialises MLState. The pipeline tree widget populates with all stage nodes in ⬜ pending state.

Inter-Layer Communication
How Textual, LangGraph, and Groq talk to each other without blocking
⚡
Groq Streaming → RichLog
Every Groq call uses stream=True. The async generator yields token deltas. Each delta calls RichLog.write(token) in-place, producing a typewriter effect. The LangGraph node awaits the full stream before advancing state.
🔄
LangGraph astream → Textual Messages
graph.astream(state) yields {node_name: updated_state} after each node. An on_mount background task iterates this generator and posts custom Textual Message events to update the pipeline tree, status icons, and right panel stats.
🪢
User Input → asyncio.Queue
When a node needs user input, it sets state.awaiting_input = True and blocks on await queue.get(). Textual detects the flag, reveals the bottom Input bar, and on submit posts the answer into the queue. The node unblocks and continues.
🧵
GA Thread → Textual Worker
DEAP runs synchronously in run_worker(thread=True). After each generation the callback uses call_soon_threadsafe to post a GAProgress Textual Message. The app's handler updates the Sparkline widget without blocking the async event loop.

Each agent is a LangGraph node with a Groq system prompt, structured JSON output schema, and specific Textual widgets it drives

LangGraph Node · Groq JSON mode · plotext
System Prompt Role
Senior data scientist performing EDA. Given serialised DataFrame summary, output structured findings.

🧹
Outlier Handler Node
Conditional LangGraph Node · Groq · pandas
Activation Condition
Edge condition checks len(state.outlier_indices) > 0. If false, this node is skipped entirely and the graph proceeds to Feature Engineering.
Strategy Options (user selects)
①Remove rows drop outlier indices from df
②IQR Cap clip at Q1-1.5×IQR / Q3+1.5×IQR
③Log Transform np.log1p on flagged columns
④Flag Feature add binary is_outlier_col column
⑤Keep ignore and proceed
Textual Widgets Used
RadioSet (strategy pick) DataTable (before/after diff) Static (box plot)
⚙️
Feature Engineering Agent
LangGraph Node · Groq · sandboxed eval
System Prompt Role
ML feature engineer. Given column names, dtypes, and EDA insights, propose derived features that could improve model signal. Output only safe, vectorised pandas expressions.
JSON Output Schema
→features[]
namenew column name string
exprpandas expression (safe eval)
rationalewhy this feature adds signal
Safety Sandbox
Expressions are evaluated inside a restricted eval() with only df, np, and pd in scope. No imports, no builtins. Failed evals are caught, logged as warnings, and skipped.
Textual Widgets Used
DataTable (live schema) RichLog (feature log) Checkbox (approve each)

Widget-to-Textual Class Mapping
🌲
Left Panel — Tree
Textual Tree widget with custom TreeNode data. Status icons updated by posting UpdateNodeStatus messages from the LangGraph stream handler. Supports keyboard navigation.
📜
Center Log — RichLog
Textual RichLog with markup=True. Groq token stream writes directly into it via write() in the async generator loop. Charts injected as Static(renderable) mounts.
📊
Charts — Static + plotext
plotext renders to a string buffer via plt.build(). The string is wrapped in a Rich Text object and mounted as a Static widget inside the center scroll area. Fully native.
⚡
Right Stats — DataTable + Sparkline
Textual DataTable for the data preview, Sparkline for fitness curve, custom Grid layout for stat rows. All updated reactively via reactive() descriptors when MLState changes.
⌨️
Bottom Input Bar
Textual Input widget hidden via CSS display:none by default. When state.awaiting_input = True, a reveal_input(prompt) method sets CSS, updates label, focuses input. On submit, calls queue.put().
🎛️
Modal Dialogs
Textual ModalScreen subclass for: strategy selection (outlier), GA config form, final results summary. Pushed via app.push_screen() and dismissed with result callback.

DEAP Setup — ga_trainer_node.py (conceptual)
# Chromosome: list of gene indices GENE_SPACES = { "n_estimators": list(range(10, 501, 10)), "max_depth": [None] + list(range(2, 31)), "max_features": ["sqrt", "log2", 0.3, 0.5], "criterion": ["gini", "entropy"], ...8 genes total for RF... } creator.create("FitnessMax", base.Fitness, weights=(1.0,)) creator.create("Individual", list, fitness=creator.FitnessMax) def decode_chromosome(ind): return { k: GENE_SPACES[k][ind[i]] for i, k in enumerate(GENE_SPACES) } def evaluate(individual): params = decode_chromosome(individual) model = RandomForestClassifier(**params) scores = cross_val_score( model, X_train, y_train, cv=3 ) return (scores.mean(),) # tuple for DEAP # After each generation — post to TUI def gen_callback(gen, best, avg): loop.call_soon_threadsafe( q.put_nowait, GAProgress(gen, best, avg) )
Evolutionary Loop
① Initialise Population
50 random chromosomes. Each gene sampled uniformly from its space. Diverse initial population avoids local optima.
▼
② Evaluate Fitness
3-fold CV on training split. Parallelised via joblib with n_jobs=-1. Results cached by chromosome hash to avoid re-evaluating identical individuals.
▼
③ Selection
Tournament selection (size=3). 3 individuals drawn at random; fittest survives. Preserves diversity better than roulette wheel at small populations.
▼
④ Crossover + Mutation
Two-point crossover (p=0.7) swaps gene segments. Uniform integer mutation (p=0.2) randomly reassigns individual genes. Hall of Fame preserves top-5 elites across generations.
▼
⑤ Post Progress → TUI
gen_callback fires. Textual Sparkline + DataTable update. If N generations complete or fitness plateau detected (Δ < 0.001 for 5 gens), convergence declared.
📡
↺ repeat
⑥ Best Chromosome → Retrain
Hall of Fame champion decoded to params. Model retrained on full training set (not just CV fold). Passed to Evaluator node. Chromosome saved to state for reproducibility.
